# 1. 学習塾の新規生徒情報を自動同期するシステム
2022年5月 - 現在

## 概要
自社開発している学習塾の営業業務を支援するWEBシステムから、導入先である学習塾の基幹システムへ、生徒情報を自動同期させるシステム。

## 開発手法
- 開発工程ごとにGitHub Issueを作成し、検証や実装を行う。
- いくつかの工程をまとめた状態をデプロイし、機能を検証するを繰り返す。

## 開発チーム
- 私(メインで開発)
- 上長(サポート)

## 主な使用技術
- JavaScript
- Node.js
- Playwright
- Serverless
- AWS Lambda
- AWS DynamoDB

## 担当業務
システムの新規開発から、自社で持つ学習塾への導入までのフェーズを担当。
- 営業チームと議論しながら、簡易的なシステム構成図やシーケンス図を作成
- 使用技術選定
- 実装
- インフラ環境構築
- デプロイ
- 仕様書作成
- ユーザーへの導入資料作成

## 学び
### 手を早く動かす大切さ
- 選定した技術で、やりたいことが実現可能かどうかを検証する必要がありました。
- 検証作業を早く繰り返し、実現性の確認と作業工程の見積もりを早期に行えました。
- 検討事項も明確にでき、営業チームと議論を重ねるゆとりが生まれ、スムーズな実装ができました。

# 2. 学習塾の運営業務を効率化するWEBシステム
- 前半：2020年7月 - 2021年3月
- 後半：2021年7月 - 2022年4月

## 開発手法
- 開発項目(新規機能開発、バグ改修など)ごとにGitHub Issueを作成し、実装を行う。
- 1週間を1スプリントとし、1スプリントごとにデプロイを行う。

## 開発チーム
前半
- 私 (開発メンバー)
- 上長 (プロジェクトリーダー)
- 外国人エンジニア6-8名 (開発メンバー)

後半
- 私 (開発メンバーリーダー)
- 上長 (プロジェクトリーダー)
- 外国人エンジニア2-3名 (開発メンバー)

## 主な使用技術
- Docker
- PHP
- Laravel
- MySQL
- Bootstrap
- CoreUI
- JavaScript
- jQuery
- Node.js
- Cypress
- AWS (ECR、ECS、RDS、CloudWatch、EC2)
- Sentry

## 担当業務
### 新規機能開発とバグ改修
システムの追加開発から、自社で持つ学習塾での運用までのフェーズを担当。
- 営業チームと議論しながら
    - 仕様の整理、UI設計
    - 開発チーム用の仕様書作成と修正
- Local、Staging、Production環境でのバグ調査
- 実装 (単体テスト、結合テスト、EndToEndテスト実装含む)
- 手動テスト項目作成と修正

### 運用・保守
- Staging、Production環境へのデプロイ作業
- 脆弱性パッケージ更新

### チームマネジメント
- チームメンバーのタスク管理
- チームメンバーのプルリクエストをレビュー
- 月1でチームメンバーと1on1

## 取り組み
### 散在する脆弱・膨大・複雑ロジックを積極的リファクタリング
- タスクに関連したロジックは、型定義、メソッド抽出、クラス最小化などを積極的に行いました。
- タスク外のロジックには、コメント (TODO、FIXME)を残すようにしました。
- リファクタリングの際は、テストコードの追加と更新を行い、バグを極力排除しました。
- また、レビューやslackチャットを通し、チームメンバーに対しても積極的なリファクタリングを促しました。
- 機能追加やバグ改修の障壁を低くし、後続タスクの時間を軽減することができました。

### 学びをチームへ共有、レビューの負担を軽減
- 私が作成したプルリクエスト(PR)は上長がレビューしていました。
- 他メンバーが作成したPRの1stレビューは私が、最終レビューを上長が担当していました。
- レビューやslackチャットを通して、上長によるレビューから得た学びなどをチームメンバーへ共有しました。
- その結果、チームの実装スキルが向上し、レビューにかかる時間が短縮しました。
- 特に上長による最終レビューでは、一発LGTMが頻繁に出るようになりました。

# 自己PR
## 1. ただ仕様通りに実装するだけじゃない
- こうしたらもっとユーザーにとって使いやすくなるのではないか
- 大学と前職で培ったUIUX視点での提案も行えることが強みです。
- 現職では、タスク範囲内外関わらず積極的に提案をし、UIUXの改善に貢献しました。
- 1つの改善に対し、複数案とそれぞれのメリットデメリットなどをまとめ、提案していました。

## 2. 高い自己研鑽力
- リモートワーク下でも、外国人エンジニアとコミュニケーションがスムーズに取れるように、
- 業務時間外で1時間程度確保し、英語を勉強しました。
- また、業務、Udemy、書籍などから得た学びを、til(today i learned)としてGitHubリポジトリにまとめていました。
- そのtilを、レビューや日々のチャットを通してチームメンバーに共有し、チームのスキルアップも図りました。

## 3. フルリモート下でのコミュニケーション力
- 未経験WEBエンジニアとして入社してから現在に至るまで、フルリモートで働いていました。
- 業務では、slackでのチャットと通話を使っており、以下のことを意識していました。
- slackチャットでは、
    - 質問や共有をする際は、背景と目的を必要に応じて記載。
    - 提案をする際には、複数の提案とそれぞれにおけるメリットデメリット、おすすめ案を記載。
    - 参考リンクや画像は、分かりやすくなる場合に必ず記載。
- slack通話では、
    - 通話で議論すべき内容を事前にまとめ、通話前にチャットで送信。
    - 議事録はチャット上でリアルタイムでまとめ、不参加書も含めて確認できるように。
