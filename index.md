# 自己PR
## 1. ただコードを書くだけじゃない。
- こうしたらもっとユーザーにとって使いやすくなるのではないかという
- 大学と前職(デザイナー) で培った UI / UX視点での提案を行えることが強みです。
- 現職で携わったプロジェクトでは、タスク範囲内外関わらず積極的に提案をし、UI / UXの改善に貢献しました。
- 提案は
    - 1つの提案に対し、複数案とそれぞれのメリットデメリットなどをまとめ、
    - その後チームと議論を重ね、提案が通ればGitHub Issueを作成する流れで行っていました。

## 2. 働く環境や技術に対するキャッチアップの姿勢
- 外国人エンジニアとコミュニケーションを取る時は英語を使います。
- リモートワークでのコミュニケーションコストを削減するため、業務時間外で英語を勉強しました。
- また、業務 / Udemy / 本などから得た学びを、til(today i learned)としてまとめて、スキルアップをしました。
- そのtilを外国人エンジニアへ積極的に共有し、チームのスキルアップも図りました。

## 3. フルリモートでも業務を問題なく行えます。
- 未経験 WEB エンジニアとして入社してから現在に至るまで、フルリモートで働いていました。
- slackでのチャットと通話を使って、コミュニケーションを取っていました。
- slackチャットでは、
    - 質問や共有をする際は、背景と目的を必要に応じて記載。
    - 提案をする際には、複数の提案とそれぞれにおけるメリットデメリット、おすすめの解決策を記載。
    - 参考リンクや画像は、分かりやすくなる場合に必ず記載。
- slack 通話では、
    - 通話で議論すべき内容を事前にまとめ、通話前にチャットで送信。
    - 議事録はチャット上でリアルタイムでまとめ、不参加書も含めて確認できるように。
- 以上のことを意識し、リモートワークでも問題なくスムーズに業務を行うことができました。
